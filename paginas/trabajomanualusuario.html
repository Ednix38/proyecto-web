Proyecto Final — PokeHub

Fecha de entrega: 5 de diciembre de 2025 • Tipo: Proyecto práctico • Unidad IV

Descripción General

El proyecto final consistió en el desarrollo completo de una aplicación web llamada PokeHub, creada utilizando HTML, CSS y JavaScript. Esta plataforma integra diversas funcionalidades relacionadas con el universo Pokémon, permitiendo al usuario consultar información, interactuar con herramientas dinámicas y disfrutar de minijuegos en tiempo real.

El proyecto está diseñado como una aplicación web modular, organizada en diferentes vistas y componentes reutilizables. Se emplearon técnicas modernas de desarrollo front-end, manejo de datos con la PokeAPI, y programación orientada a eventos para construir una experiencia fluida, intuitiva y visualmente atractiva.

Objetivo del Proyecto

Aplicar los conocimientos obtenidos durante la materia para construir una aplicación web funcional y completa que combine interfaces visuales, interactividad avanzada y consumo de APIs externas. Además, se buscó que el proyecto reflejara buenas prácticas de estructuración, limpieza de código, accesibilidad y diseño responsivo.

Características del Proyecto

1. Pokédex Interactiva

  • Consulta dinámica de más de 1,000 Pokémon usando PokeAPI.
  • Tarjetas con información detallada: tipos, estadísticas, habilidades e imágenes oficiales.
  • Sistema de filtros por nombre, tipo y generación.
  • Carga automática y diseño responsivo.

2. Simulador de Batallas

Una de las funciones principales del proyecto consiste en simular combates Pokémon.

  • Selección de dos equipos personalizados de hasta 6 Pokémon.
  • Sistema de turnos inspirado en Pokémon Showdown.
  • Movimientos con daño real calculado según estadísticas, tipos y efectos.
  • Animaciones visuales para mayor inmersión.

3. Calculadora Avanzada de Daño

  • Cálculo según fórmulas oficiales de la franquicia.
  • Variables: nivel, ataque, defensa, STAB, clima, crítico, tipos y más.
  • Interfaz clara y amigable para comparar resultados entre movimientos.

4. Herramientas de IVs y EVs

  • Calculadora de IVs a partir de estadísticas reales.
  • Planificador de EVs para builds competitivas.
  • Representación gráfica para mayor comprensión.

5. Minijuego: "Who's That Pokémon?"

  • El clásico reto de adivinar Pokémon basándose en su silueta.
  • Imágenes ocultas con filtros CSS para generar misterio.
  • Puntuación, intentos y selector de dificultad.

6. Diseño y Arquitectura

  • Interfaz construida con una estructura modular escalable.
  • Sistema de navegación SPA (Single Page Application) sin recargas completas.
  • Estilos modernos con CSS: dark mode, tarjetas, contenedores, sombras y glass-morphism sutil.
  • Archivos JS bien divididos: pokédex, batalla, calculadora y utilidades.

Proceso de Desarrollo

El proyecto se realizó en fases: diseño básico, maquetación, consumo de API, programación de lógica interna, tests de funcionalidad y optimización visual. Se emplearon herramientas como DevTools, Git, extensiones de VSCode y validadores de HTML y CSS.

Una parte clave del desarrollo fue comprender el manejo de datos asíncronos, promesas y fetch(), así como la integración de componentes que dependen unos de otros dentro de una SPA.

Conclusiones

Este proyecto final permitió consolidar conocimientos esenciales del desarrollo web moderno mediante la creación de una aplicación completa, funcional y visualmente sólida. Se logró combinar programación, diseño, consumo de APIs, manejo de eventos e interacción avanzada, lo que refleja un aprendizaje significativo y un crecimiento real en el dominio del front-end.

PokeHub no solo funciona como proyecto académico, sino como una base sólida para futuras mejoras, integración de bases de datos, backend e incluso expansión hacia aplicaciones móviles.
